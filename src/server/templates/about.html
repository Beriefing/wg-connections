WG-connections

This is a demo page for the WG-connections GitHub project.

It helps you search for WGs (flatshares) or flats by showing the commute time from your POI (work, university,...) to each offer.

The project consists of 3 different parts: Scraper, Server and Router.

Scraper:

The scraper uses Selenium, BeautifulSoup and a rotating-proxy service to scrape data from a certain big German flatsharing website. Currently only data for Berlin is scraped.

Server:

The server is based on Flask and uses a Leaflet map to draw the geojson elements returned from a running GraphHopper instance.

Router:

There need to be a couple of running (selfhosted) GraphHopper instances with map and GTFS data from Berlin. The setup is as easy as shown in the examples on their project page.

Graphhopper Instances:
  1. Public Transit (very RAM-intensive), Port: 8993
  2. Car, Bike, Walk share one instance, Port: 8989
  3. Isochrones (for Car, Bike + Walk), Port: 8991
